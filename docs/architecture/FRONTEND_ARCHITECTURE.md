# Frontend Architecture

## Purpose

This document describes the frontend architecture of the **Power Electronics Reliability Copilot**.

The frontend provides the primary user interface for interacting with the backend services, allowing engineers to upload engineering documents, manage document processing workflows, and, in future releases, interact with the Engineering Copilot through a conversational interface.

The frontend is intentionally lightweight, with business logic and AI processing delegated to the backend.

---

# Overview

The frontend is implemented as a modern single-page application (SPA) using **React**, **TypeScript**, and **Vite**.

Its primary responsibilities are:

- presenting a responsive user interface
- managing user interactions
- communicating with backend REST APIs
- displaying processing results
- visualising engineering evidence

The frontend does **not** perform engineering reasoning or knowledge processing. These responsibilities remain within the backend services.

---

# Design Principles

The frontend architecture follows several key principles.

## Thin Client

The frontend focuses on presentation and user interaction.

Engineering intelligence remains in the backend.

---

## API-driven

All communication with the backend occurs through REST APIs.

This ensures a clear separation between the user interface and backend services.

---

## Modular Components

The user interface is composed of reusable React components.

This allows new capabilities to be introduced without restructuring the entire application.

---

## Extensibility

The frontend has been designed to support future capabilities, including conversational AI, evidence visualisation, and Knowledge Graph exploration.

---

# Frontend Architecture

The current frontend architecture is shown below.

```text
                User
                 │
                 ▼
      React + TypeScript UI
                 │
                 ▼
      Frontend Components
                 │
                 ▼
        REST API Client
                 │
                 ▼
         FastAPI Backend
```

The frontend acts as the presentation layer while all engineering processing occurs within the backend.

# Current Components

Version **0.5** currently provides the following user interface components.

## Document Upload

Allows engineers to upload technical documents for processing.

Supported document types include:

- PDF
- TXT
- CSV

---

## Document Management

Provides access to registered engineering documents and their processing status.

---

## Backend Connectivity

Communicates with the backend through REST APIs documented using Swagger/OpenAPI.

Current interactions include:

- document upload
- document registration
- processing requests

---

## Engineering Results

The frontend has been prepared to display engineering responses generated by the backend.

Current focus is on integrating backend services rather than implementing conversational workflows.

---

# Future Evolution

The next stage of frontend development is planned for **Version 0.5.1**.

Planned enhancements include:

## Conversational Engineering Copilot

A chat-based interface allowing engineers to ask technical questions directly.

Example:

```text
Engineer

"Why is VCE(sat) increasing during power cycling?"
```

The frontend will submit the question to the Engineering Copilot backend and display:

- engineering answer
- supporting document evidence
- Knowledge Graph reasoning
- confidence assessment
- recommended next steps

---

## Engineering Evidence Panel

Display retrieved engineering evidence including:

- semantic document chunks
- similarity scores
- source documents
- evidence metadata

---

## Knowledge Graph Visualisation

Interactive exploration of:

- engineering entities
- relationships
- evidence links
- diagnostic pathways

---

## Conversation History

Support multi-turn engineering discussions by preserving previous questions and responses during an investigation.
