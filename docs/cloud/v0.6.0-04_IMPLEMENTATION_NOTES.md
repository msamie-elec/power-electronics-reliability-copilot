# Azure Configuration Additions

Added to Azure infrastructure configuration:

```env
LOG_ANALYTICS_WORKSPACE=law-powerelec-copilot-dev
APPLICATION_INSIGHTS_NAME=appi-powerelec-copilot-dev
```

These values are non-secret Azure resource names and belong in:

```text
infra/azure/env/azure.local.env
```

They should not be placed in:

```text
infra/azure/env/azure.secrets.local.env
```

# Health Diagnostics

The backend now provides:


`/health/details` validates:

- Azure Key Vault
- Azure Blob Storage
- Azure OpenAI configuration
- Neo4j
- Provider configuration
- Dependency latency
- Uptime

# Notes

## Swagger refresh

Swagger UI may cache OpenAPI metadata in the browser. If the displayed version does not refresh after restarting the backend, close and reopen the Swagger page or inspect:

```text
http://127.0.0.1:8000/openapi.json
```

## Application Insights data

The Application Insights resource exists and is ready to receive telemetry. It may show limited graphs during local development because the backend has not yet been deployed as a continuously running Azure service.

Runtime telemetry becomes more useful after v0.7.0 production hosting introduces persistent Azure-hosted backend traffic.

## Storage connection string refresh

If the Azure Storage Account is recreated, the storage connection string changes. In v0.6.0, update the local secret file and re-import secrets into Key Vault.

In v0.7.0, storage access should move to Managed Identity and Azure RBAC to remove the storage connection string dependency.
