# Documentation Style Guide

## Purpose

This document defines the documentation standards used throughout the Power Electronics Reliability Copilot repository.

---

# Writing Principles

Documentation should be:

- Clear
- Concise
- Technically accurate
- Version controlled
- Easy to maintain

Write for engineers rather than for academic publication.

---

# Headings

Use the following hierarchy.

# Title

## Major Section

### Subsection

#### Detail

---

# Paragraphs

Keep paragraphs relatively short.

Explain:

- what
- why
- how

Avoid unnecessary repetition.

---

# Lists

Use bullet lists for concepts.

Use numbered lists for procedures.

---

# Tables

Prefer tables when comparing concepts.

Example

| Item | Description |
|------|-------------|

---

# Diagrams

Every diagram should include:

- Purpose
- Description
- File location
- Version

---

# Code Blocks

Always specify language.

```cypher
MATCH (n)
RETURN n;
```

---

# File Naming

Use

UPPER_CASE

for major architecture documents.

Use

snake_case

for implementation files.

---

# Versioning

Every significant document should contain:

Version

Last Updated

Author (optional)

Related Documents
