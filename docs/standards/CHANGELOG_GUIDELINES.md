# Changelog Guidelines

## Purpose

Defines how project changes should be documented.

---

# Version Format

Major.Minor.Patch

Example

v0.4.0

---

# Release Structure

## Added

New features.

---

## Changed

Updated functionality.

---

## Fixed

Bug fixes.

---

## Removed

Deprecated functionality.

---

## Documentation

New or updated documentation.

---
