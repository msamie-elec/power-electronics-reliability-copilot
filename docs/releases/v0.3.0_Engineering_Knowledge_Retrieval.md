# Then comes the really exciting part...

## Release v0.4.0 — GraphRAG & Engineering Knowledge Graph

This is where your project starts to stand out from most RAG applications.

Instead of retrieving only by vector similarity:

```
Question

↓

Embedding

↓

Nearest Chunks
```

you'll retrieve using both semantic similarity and engineering relationships:

```
Question
      │
      ▼
Embedding Search
      │
      ▼
Relevant Chunks
      │
      ▼
Neo4j Knowledge Graph
      │
      ▼
Related Components
      │
      ▼
Failure Mechanisms
      │
      ▼
Maintenance History
      │
      ▼
Combined Context
      │
      ▼
LLM
```

For example, a question about an **IGBT module** won't just retrieve chunks mentioning "IGBT". The graph can also traverse to related concepts like:

```
IGBT Module
        │
        ├── uses
        ▼
Bond Wire
        │
        ├── suffers_from
        ▼
Thermal Fatigue
        │
        ├── causes
        ▼
Intermittent Failure
        │
        ├── mitigated_by
        ▼
Maintenance Action
```

That graph-enhanced reasoning is much closer to how an experienced reliability engineer thinks.
