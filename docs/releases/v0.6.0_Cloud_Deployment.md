# Version 0.6.0 — Cloud-Native Azure Deployment

## Status

**In Progress**

---

# Overview

Version **0.6.0** transitions the Power Electronics Reliability Copilot from a locally executed engineering application into a cloud-native AI engineering platform hosted on Microsoft Azure.

Previous releases established the Engineering Copilot, Retrieval-Augmented Generation (RAG), Knowledge Graph integration, evidence-backed reasoning, semantic retrieval, and the professional engineering workspace.

Version **0.6.0** focuses on cloud-native deployment while preserving the existing engineering architecture. During implementation, the document ingestion workflow was also modernised so that newly uploaded engineering documents automatically become searchable without manual preprocessing.

The objective of this release is to demonstrate enterprise cloud engineering practices while preserving the trustworthy evidence-backed reasoning architecture developed throughout Versions **0.1.0–0.5.2**.

---

# Objectives

The primary objectives of Version **0.6.0** are to:

- Deploy the application to Microsoft Azure.
- Introduce cloud-native architecture.
- Replace local configuration with cloud configuration.
- Secure application secrets using Azure Key Vault.
- Store engineering documents using Azure Blob Storage.
- Integrate Azure OpenAI.
- Introduce cloud monitoring and diagnostics.
- Eliminate manual knowledge-index generation after document upload.
- Prepare the application for production engineering.

## Automatic Engineering Knowledge Pipeline

One of the major improvements introduced during Version **0.6.0** is automatic document preparation.

Previously the engineering knowledge pipeline required multiple manual execution steps after every uploaded document.

The workflow is now fully automated.

After a PDF upload the backend automatically performs:

```
PDF Upload
      ↓
Document Registration
      ↓
PDF Parsing
      ↓
Knowledge Chunk Generation
      ↓
Embedding Generation
      ↓
FAISS Index Generation
      ↓
Semantic Retrieval Ready
```

Implemented:

- Automatic deterministic document ID generation
- Automatic document registration
- Automatic PDF parsing
- Automatic engineering chunk generation
- Automatic embedding generation
- Automatic FAISS index generation
- Automatic vector mapping generation

This allows every newly uploaded engineering document to become immediately searchable by the Engineering Copilot.

# Release Summary

Version **0.6.0** represents the transition from a locally executed engineering application to a cloud-native AI engineering platform.

Beyond introducing Microsoft Azure services, this release modernises the engineering document ingestion pipeline through fully automated document registration, semantic indexing, vector generation, and Retrieval-Augmented Generation preparation.

The Engineering Copilot can now ingest a newly uploaded engineering document and automatically prepare it for evidence-backed reasoning without requiring any manual preprocessing steps, providing a significantly more scalable and production-oriented architecture.
