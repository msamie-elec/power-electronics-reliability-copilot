# Version 0.7.0 — Production Engineering & DevOps

## Status

**Planned**

---

# Overview

Version **0.7.0** transforms the cloud-ready architecture introduced in Version **0.6.0** into a fully hosted, production-oriented engineering platform.

While Version **0.6.0** integrated Microsoft Azure services such as Azure OpenAI and Azure Blob Storage, the frontend and backend applications continued to execute locally. Version **0.7.0** completes the transition by deploying the application itself to Microsoft Azure using modern containerisation, cloud hosting and DevOps practices.

This release focuses on production deployment, automation, operational reliability and enterprise software engineering. Rather than introducing new engineering intelligence capabilities, Version **0.7.0** demonstrates how an AI application can be securely deployed, operated and maintained within a modern cloud environment.

---

# Objectives

The primary objectives of Version **0.7.0** are to:

- Deploy the backend application to Microsoft Azure.
- Deploy the frontend application to Microsoft Azure.
- Containerise the backend using Docker.
- Introduce production deployment automation.
- Build CI/CD pipelines.
- Improve operational reliability.
- Introduce production monitoring and diagnostics.
- Prepare the platform for enterprise-scale deployment.

---

# Major Capabilities

## Azure Container Apps Deployment

The FastAPI backend will be deployed to Azure Container Apps.

Deployment provides:

- managed container hosting
- automatic HTTPS
- cloud scalability
- production networking
- secure Azure integration

The backend will continue to consume Azure OpenAI, Azure Blob Storage and other Azure services introduced in Version **0.6.0**.

---

## Azure Static Web Apps

The React engineering workspace will be deployed using Azure Static Web Apps.

Deployment includes:

- production frontend hosting
- global content delivery
- secure HTTPS
- cloud-based frontend deployment

The engineering workspace becomes accessible through a public Azure-hosted endpoint rather than local development servers.

---

## Docker Containerisation

The backend application will be containerised using production-ready Docker images.

Containerisation provides:

- reproducible deployments
- environment consistency
- simplified cloud hosting
- portable runtime environments

Container images will form the foundation for future Kubernetes deployment.

---

## Azure Container Registry

Container images will be stored within Azure Container Registry.

This provides:

- secure image storage
- version management
- Azure-native deployment integration
- deployment consistency

---

## Continuous Integration & Continuous Deployment

Deployment automation will be introduced through GitHub Actions.

The CI/CD pipeline will include:

- source checkout
- dependency installation
- automated testing
- image build
- image publishing
- Azure deployment
- deployment verification

This enables repeatable production deployments directly from the GitHub repository.

---

## Azure Key Vault Integration

The preparation completed in Version **0.6.0** will be finalised.

Sensitive configuration will be managed through Azure Key Vault, including:

- Azure OpenAI credentials
- Storage credentials
- application secrets
- deployment configuration

The application will no longer rely on locally stored production secrets.

---

## Azure Monitor & Application Insights

Production monitoring will be introduced using Azure Monitor and Application Insights.

Operational visibility will include:

- request tracing
- application logs
- exception reporting
- performance metrics
- dependency monitoring
- availability monitoring

These capabilities provide operational insight into production behaviour.

---

## Operational Diagnostics

Production diagnostics will be expanded through enhanced health endpoints and cloud monitoring.

Capabilities include:

- application health
- dependency status
- Azure service connectivity
- runtime information
- deployment diagnostics

---

## Production Deployment Workflow

Version **0.7.0** establishes a complete deployment workflow from development through production.

The workflow includes:

- local development
- automated testing
- Docker image creation
- Azure image publishing
- cloud deployment
- deployment validation
- production monitoring

# Known Limitations

The following capabilities remain outside the scope of Version **0.7.0**:

- Kubernetes orchestration (AKS)
- horizontal autoscaling
- production load balancing
- multi-region deployment
- disaster recovery
- advanced infrastructure-as-code
- enterprise governance policies

These capabilities may be introduced after Version **1.0.0** depending on future project direction.
