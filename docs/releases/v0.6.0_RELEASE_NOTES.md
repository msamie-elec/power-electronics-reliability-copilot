# v0.6.0 Release Notes — Cloud-Native Azure Integration

## Release Status

**Version:** v0.6.0  
**Status:** Completed  
**Theme:** Cloud-native Azure integration, secure secret management, automated engineering knowledge ingestion, observability and release validation.

---

# Summary

Version **v0.6.0** establishes the cloud-native foundation for the Power Electronics Reliability Copilot.

This release integrates the existing engineering intelligence platform with Microsoft Azure services while preserving the provider-based architecture introduced in earlier versions.

The release adds:

- Azure Blob Storage integration.
- Azure OpenAI integration.
- Azure Key Vault secret management.
- Azure Monitor and Application Insights foundation.
- Automatic engineering document indexing after upload.
- Production-style health diagnostics.
- Azure infrastructure automation and validation scripts.

Frontend and backend cloud hosting are intentionally deferred to **v0.7.0**.

---

# Major Capabilities

## Cloud-Native Configuration

The backend now supports cloud-aware configuration for Azure services while preserving local development support.

## Secret Management

The application now uses a provider-based secret architecture:

```text
SecretService
├── LocalSecretProvider
└── AzureKeyVaultSecretProvider
```

Azure OpenAI, Azure Blob Storage and Neo4j credentials can be retrieved through Azure Key Vault.

## Azure Blob Storage

Uploaded engineering documents can be persisted in Azure Blob Storage through the storage provider abstraction.

## Azure OpenAI

The AI provider layer supports Azure OpenAI deployments for engineering reasoning and embeddings.

## Automated Knowledge Ingestion

Newly uploaded engineering documents are automatically processed into searchable knowledge assets:

```text
Upload
→ Document registration
→ PDF parsing
→ Chunking
→ Embedding generation
→ FAISS indexing
→ Semantic retrieval readiness
```

## Observability

v0.6.0 introduces Azure observability foundations:

- Log Analytics Workspace
- Application Insights
- Monitoring validation script
- Application Insights connection string stored in Key Vault
- Backend health diagnostics

## Health Diagnostics

The backend now exposes:

```text
GET /health
GET /health/details
```

The detailed endpoint validates Azure Key Vault, Azure Blob Storage, Azure OpenAI configuration and Neo4j connectivity.

# Known Notes

## Application Insights Portal

Application Insights resources are available, but portal dashboards may show limited data until the backend is deployed and receives regular traffic.

## Azure Storage Connection String

In v0.6.0, Azure Blob Storage uses a connection string stored in Azure Key Vault.

In v0.7.0, this should be replaced with Managed Identity and Azure RBAC-based Blob Storage access.

## Hosting

Backend and frontend Azure hosting are not part of v0.6.0. They are planned for v0.7.0.
